    Org-babel python table header

    serycjon

    2017-06-30 11:18

    Comments

    Recently, I finally found a good use-case for python code block in an org-mode file. My goal was to generate a table, which wasn't hard at all: 

    #+BEGIN_SRC python :results table :exports results
  table = []
  for i in range(4):
      table.append([i, 2**i])
  return table
#+END_SRC
    0 1
    1 2
    2 4
    3 8

    Nice, but how to add a header (give each column a name)? It is quite surprising, how hard it is to find any relevant info about that. After some elisp research, I have discovered org-babel-python-None-to variable, which defaults to hline. With that set, it is now easy to add header separator by simply inserting None into the list returned by the python code as in the following example. 

    #+BEGIN_SRC python :results table :exports results
  table = [['#n', 'n-th power of 2'], None]
  for i in range(4):
      table.append([i, 2**i])
  return table
#+END_SRC
    #n n-th power of 2
    0 1
    1 2
    2 4
    3 8

    BTW I have added the following setting in the Nikola orgmode plugin init.el in order to style the tables nicely. 

    (setq org-html-table-default-attributes
      (quote
       (:border "2" :cellspacing "0" :cellpadding "6" :rules "groups"
		:frame "hsides" :class "table-condensed table-bordered")))

    Blogging with Nikola

    serycjon

    2017-06-29 15:13

    Comments

    I have been using blogger.com for posting photos and stories about my travels. Recently I have done some experiments with Jekyll (jekyllrb.com) - a ruby based static site generator. Althought it is possible to get it working with org-mode, it didn't feel like I will stuck to it.

    A few days ago, this post about blogging with Emacs org-mode and Nikola was published and I decided I will give it a try.

    Nikola is a static site generator written in python with many neat features, here is how I've got it running.

    Basic Nikola installation

    virtualenv

    I have never used python virtual environments and installed all the packages globally, but since I use python 2.7 and Nikola wants python 3, I gave it a try.

    Install virtualenv and virtualenvwrapper 

    $ sudo apt install virtualenv virtualenvwrapper
$ mkdir ~/.virtualenvs

    Set some variables and source the virtualenvwrapper - add this to .zshrc (or .bashrc). 

    <span style="font-weight: bold;">export</span> <span style="font-weight: bold; font-style: italic;">WORKON_HOME</span>=$<span style="font-weight: bold; font-style: italic;">HOME</span>/.virtualenvs
<span style="font-weight: bold;">export</span> <span style="font-weight: bold; font-style: italic;">PROJECT_HOME</span>=$<span style="font-weight: bold; font-style: italic;">HOME</span>/dev
<span style="font-weight: bold;">source</span> /usr/share/virtualenvwrapper/virtualenvwrapper_lazy.sh

    Don't forget to restart the shell or source the .zshrc afterwards.

    Now you are ready to create the virtual environment for your blog: 

    $ mkvirtualenv --python=/usr/bin/python3 blog

    After this, it is possible to get inside this environment with workon blog and get out of it with deactivate.

    First steps with Nikola

    Getting started with Nikola is just extremely simple. First install it (and the optional extras) with 

    $ workon blog
$ pip install <span style="font-style: italic;">"Nikola[extras]"</span>

    With the engine installed, use the following command to create a directory with a new site (containing some demo content), build the blog and start a server (running on localhost:8000). 

    $ nikola init --demo blog_dir
$ nikola build
$ nikola serve -b

    The build and serve can be automated by nikola auto, which starts a server but also looks for any changes and rebuild the site if necessary.

    Getting Emacs involved

    Orgmode plugin

    At this point, you can easily start to publish your awesome content by calling 

    $ nikola new_post

    which will create a new reStructuredText file in the posts/ directory. Yet another markdown? Fortunately, there is a more comfortable way - nikola orgmode plugin.

    Install it with 

    $ nikola plugin -i orgmode

    You will then have to add some simple stuff to the site conf.py file, as described during the installation (and on the plugin web page).

    From now on, you can call 

    $ nikola new_post posts/filename.org

    to create a new blog post with pre-generated nikola metadata.

    YASnippet post template

    Org-mode with it's tables, babel and other awesome stuff already works at this point, but I didn't really like to call the new_post command, so I have taken the content it generates and put it into a YASnippet template. 

    # key: post
# name: post
# --
#+BEGIN_COMMENT
.. title: ${1:post title}
.. slug: ${1:$(replace-regexp-in-string
                 <span style="font-style: italic;">"[</span><span style="font-style: italic;">^</span><span style="font-style: italic;">0-9a-z-]"</span> <span style="font-style: italic;">""</span>
           (replace-regexp-in-string
             <span style="font-style: italic;">" "</span> <span style="font-style: italic;">"-"</span>
             (downcase yas-text)))<span style="font-weight: bold;">}</span>
.. date: `(format-time-string <span style="font-style: italic;">"%Y-%m-%d %H:%M:%S %Z%z"</span>)`
.. tags:
.. category:
.. link:
.. description:
.. type: text
#+END_COMMENT

$0

    This way, I can just create new posts/filename.org, write post, press <TAB> to expand the snippet and I am good to go. The template even parses the post title and automatically creates a pretty url.

    Full management from Emacs

    The last slightly annoying thing is the need of switching the python environment and running nikola auto. Mike Zamansky has suggested using prodigy, which is a very nice package I have started using recently for running imapnotify and redshift (check them out, if you don't know them yet).

    The issue with Mike's solution is the need to manually set the Emacs virtualenv. I have played with it a bit and found a simple prodigy-only way. 

    (prodigy-define-service
  <span style="font-weight: bold;">:name</span> <span style="font-style: italic;">"Nikola"</span>
  <span style="font-weight: bold;">:command</span> <span style="font-style: italic;">"nikola"</span>
  <span style="font-weight: bold;">:args</span> (list <span style="font-style: italic;">"auto"</span>)
  <span style="font-weight: bold;">:cwd</span> <span style="font-style: italic;">"/path_to_the/nikola_blog/"</span>
  <span style="font-weight: bold;">:tags</span> '(nikola)
  <span style="font-weight: bold;">:init</span> '(lambda () (pyvenv-workon <span style="font-style: italic;">"blog"</span>))
  <span style="font-weight: bold;">:kill-signal</span> 'sigkill)

    Other stuff

    Blogger.com import

    You can export your whole site from blogger.com in Settings > Other > Import & back up, which creates a single xml file with all your posts. These can then be easily imported using import_blogger plugin. 

    $ nikola plugin -i import_blogger
<span style="font-weight: bold; font-style: italic;"># </span><span style="font-weight: bold; font-style: italic;">This didn't work, because of a missing dependency, so repeat after:</span>
$ pip install feedparser

<span style="font-weight: bold; font-style: italic;"># </span><span style="font-weight: bold; font-style: italic;">do this outside the site directory and copy the posts afterwards</span>
$ nikola import_blogger the_blogger_export.xml

    Github pages deploy

    Making your site public using Github pages is extremely simple, just follow the steps in the Nikola handbook: https://getnikola.com/handbook.html#deploying-to-github

Druhak (a to hlavnak) nuz (i kdyz naostreny, jak nejvic jsem dokazal) ma porad sklouzavat ve smeru vlaken dreva, coz obzvlast u oblych tvaru vazne moc nepomaha...

    Stavka a cesta domu

    O vikendu kolem 28.5. se konala oslava dedovych osmdesatin, takze jsem par tydnu pred tim koupil levne letenky a tesil se, ze budu chvilku zase doma. Do Prahy jsem mel letet ve ctvrtek a zpatky potom v utery. Par dni pred odjezdem jsem ale zjistil, ze v to utery je naplanovana v Belgii generalni stavka vseho, takze asi bude docela problem se dostat z letiste zpatky domu. Uvazoval jsem o cyklojizde, ale je to z Leuvenu kolem 70km a taky by me tam dost mozna vojaci nepustili. Navic jsem nevedel, jestli by se tam dalo kolo nechat... Takze nakonec jsem musel tuto volbu vyradit a doufat, ze se stane neco jineho zabavneho, aby bylo o cem vypravet :D. A taky jo.

    Nastal vecer, nastalo jitro, den pred odjezdem. V praci kolega oznamil, ze belgicke drahy zacinaji stavkovat uz ten den. Venku bylo asi moc hezky, tak nechteli cekat do utery. Chaos, chaos chaos. U nas jakztakz jezdil tak kazdy ctvrty vlak, u Valonu nejelo nic. Spousta belgicanu jezdi vlakem kazdy den za praci do Brusele, takze totalni kolaps.

    Ve ctvrtek jsem v praci koupil listky na letistni autobus z Brussel - jih a po tom, co me nalezite vystrasili vsichni kolegove, jsem odesel z prace asi o ctyri hodiny driv, nez jsem puvodne planoval :D. Sef mi jeste nakreslil planek, kde stavi u nadrazi ten autobus na letiste, coz se mi krute vyplatilo.

    Sel jsem na vlak do Brusele a tam to vsechno zacalo. Respektive nezacalo :D Myslel jsem, ze do Brusele se dostanu celkem v pohode a pak bude akorat problem na to letiste, ale drazni nemakacenka meli jiny nazor. Cekal jsem tam asi hodinu a pul a zacalo to byt docela napinave. Na Bruxelles-Midi jsem vybehl z vlaku a bezel, ze by i Forrest zavidel. Zastavil jsem se jenom na par vterin u vojaku, kterych jsem se zeptal, jestli jsou ty busy vazne tam, kde si myslim. No nakonec jsem tam dobehl akorat kdyz ridic zaviral zavazadlovy prostor. Uff, to byl posledni autobus, kterym jsem to mohl prijemne stihnout. Mohl... kdyby celych 60km dalnice nejelo krokem v jednom obrovskem strudlu.

    K gatu jsem dorazil asi 5 minut po zaviracce podle letenky, takze jsem mel jeste chvilku, nez to vubec otevreli a zacali poustet lidi, ale dalsim autobusem uz by mi to uletelo. Jak uz jsem asi psal, jeste pred security scanem jsem potkal Bursika u plakatku s 'retard important' :). Letela s nami i skupina chlapiku - rozlucka se svobodou jednoho z nich. Byl v zupanu, pod kterym mel jednodilne plavky a dostaval od svych kumpanu ukoly jako vyfotit se, jak zveda nekterou z nebohych spolucestujicich v naruci. Pobaveni bylo vseobecne :)

    V Praze jsem hned po pristani sedl na autobus, svezl se na Nadrazi Veleslavin a musel uznat, ze ten vlez do metra je vazne mimoradne pitomy. Pak rychle na Staromestskou a do Skautskeho Institutu, kde prave koncilo planovani tabora. Surprajz surprajz :) bohuzel uz cast Udu odesla, ale se zbytkem jsme sli jeste na pivo :)

    Byl to extremne kvalitni vecer :D. Nejdriv jsme asitak hodinu a pul bloudili Prahou a nemohli se rozhodnout, kam chceme jit a pak jsme zapadli nekam, kde meli Plzen asi za 29, nebo tak neco. Bylo to krajne podezrele, ale tak co. Cena piva se vyjasnila v zapeti, kdyz jsme dostali jidelni listek (vetsina z nas mela dost hlad, ja sileny). Normalni jidla stala tak 250, takze jsme si nasli nejlevnejsi palacinky za 80, nebo tak nejak, šalatik ( :) ), hranolky a tak. Kdyz jsme vylozili sva prani cisnici, zacala ta prava legrace, nebot nemohla prekousnout tak nizke utraty a zacala se paradne rozcilovat, ze prilohy samotne nedelaji atd... tak si jeden z nas objednal salat a jako prilohu hranolky a pak jsme si to rozdelili (nebo tak nejak). Byla uplne ruda vzteky :D. Paradne jsme se bavili... Pak nam nakonec jeste odmitala moznost zaplatit jednotlive, nebot to psala na jeden ucet, ale nakonec uznala, ze jestli chce penize, tak musi ustoupit :D

    Oslava
